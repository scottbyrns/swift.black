Zewo provides fast and reliable HTTP/HTTPS servers. Our servers are crafted with the same battle-tested HTTP parser used by nginx/node.js, so you can be sure we'll take good care of your requests. TLS/SSL is handled by OpenSSL, which is the most popular SSL implementation powering only about 2/3 of all webservers around the world. 
import HTTPServer
import HTTPSServer

let hello = Responder { request in
    return Response(body: "hello world")
}

try Server(
    reusePort: true,
    port: 80,
    responder: hello
).startInBackground()

try Server(
    reusePort: true,
    port: 443,
    certificate: "cert.pem",
    privateKey: "key.pem",
    responder: hello
).start()
You'll probably want to use a router to organize your web application. Luckily, we have an excellent Router module for that. 
import HTTPServer
import Router
import ResourcefulRoute

let v1 = Router("/v1") { route in
    route.post("/users", CreateUserResponder())
}

let app = Router(matcher: TrieRouteMatcher.self) { route in
    route.get("/hello") { _ in
        return Response(body: "hello world")
    }

    route.get("/hello/:name") { request in
    	let name = request.pathParameters["name"]!
        return Response(body: "hello \(name)")
    }

    route.compose("/api", v1)
    route.resources("/videos", VideoResources())
    route.fallback = FileResponder(path: "public")
}

try Server(app).start()
Middleware allows you to reuse code for common tasks and make your life a lot easier. 
import Zewo

let logger = LoggerMiddleware(log: Log())
let contentNegotiator = ContentNegotiationMiddleware(mediaTypes: JSONMediaType(), URLEncodedFormMediaType())
let userMapper = ContentMapperMiddleware(mappingTo: User.self)

let app = Router(middleware: contentNegotiator) { route in
    route.post("/users", middleware: userMapper) { request in
        guard let user = request.user else {
            return Response(status: .BadRequest)
        }
        let id = User.save(user)
        return Response(content: id)
    }
}

try Server(middleware: logger, responder: app).start()

Go-style concurrency

Concurrent programming without callbacks

What many have come to love with the Go programming language is its concurrency model. Go uses CSP (Communicating Sequential Processes), which Zewo brings to Swift using libmill. Say goodbye to callback hell, and meet your new favourite way of building concurrent software!

Ship your code safely

Containers for each version

Swift is a living language that improves at a fast rate. Zewo provides you with utilities, such as Docker images for each release, so you can deploy your code safely.

Zewo is a platform

Build the next generation web framework with Zewo

Thanks to Zewo’s modularity, you can pick and choose what you want to use for your own framework - whether it be Venice for concurrency, the HTTP Server, the included Trie-based router matcher for your Router, or one of our many other components.
